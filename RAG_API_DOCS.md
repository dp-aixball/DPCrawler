# DPCrawler RAG 跨服检索引擎 API 接口文档

DPCrawler 底层检索引擎已经完成**核心重构**。目前实现了“前端绝美高保真渲染”与“后端毫秒级脱水索引”的双轨制完美解耦。
为了支持**跨服务器、分布式部署**（例如大语言模型服务器在 A 机器，爬虫和知识库在 B 机器），本接口采用标准的 **HTTP RESTful API + 内置静态资源 CDN** 架构，向外部提供高精度、低噪音的增强检索（RAG）语料块及原装附件访问能力。

---

## 1. 核心检索接口：`POST /api/v1/search`

作为分布式知识库检索入口，允许外部 Agent 跨网调用。

### 1.1 输入参数 (Request Payload)

Content-Type: `application/json`

| 字段名       | 类型     | 必填 | 描述                                                                                                                                                 |
| :----------- | :------- | :--- | :--------------------------------------------------------------------------------------------------------------------------------------------------- |
| `site_name`  | `String` | 是   | 具体需要检索的站点工程名（用于限定知识库范围，避免串站）。                                                                                           |
| `query`      | `String` | 是   | 用户的自然语言查询内容，或大段的长提示词（系统具有自动泛用分词和降噪能力）。                                                                         |
| `top_k`      | `usize`  | 是   | 期望返回的最高权重文档碎片的数量上限（如 `5`）。                                                                                                     |
| `threshold`  | `f64`    | 否   | 相似度或密度的最低放行阈值（默认为 `0.0` 即不过滤），用于屏蔽低质量匹配。                                                                            |

<br>

### 1.2 输出结果 (Response Array)

返回一个包含多个 `SearchResult` 对象的 JSON 数组。由于跨服务器调用无法读取宿主机的物理盘，系统不再返回无意义的绝对路径，而是直接给出来自内置 CDN 的 **文件下载与访问 URL**。

```json
[
  {
    "filename": "docs/2026年报名简章.md", // 相对于站点根目录的文件相对路径
    "title": "2026年同等学力人员申请硕士学位报考指南", 
    "score": 45, 
    "snippet": "2026 年同等学力人员申请硕士学位外国语水平...报名流程见附件。", 
    "source_url": "https://tdxl.neea.edu.cn/notice/1", // 互联网上的原始出处
    "start_line": 24, 
    "end_line": 28, 
    "matched_block": "（二）考试报名\n符合报考资格的考生须于 3 月 16 日...", // 【核心投喂物】被动态滑窗提取出的连续上下文切片，直接作为 RAG 知识送入远程 LLM。
    
    // 【跨服核心】通过 DPCrawler 内置静态服务器（或 Nginx）暴露的远程访问链接
    "md_download_url": "http://dpcrawler-host:8080/files/SiteA/docs/2026年报名简章.md", // LLM 用来即时拉取万字全文的接口
    "html_view_url": "http://dpcrawler-host:8080/files/SiteA/html_views/2026年报名简章.html" // 前端 UI 直接塞入 Iframe 画布的高保真远端视觉链接
  }
]
```

## 2. 远端渲染高保真回显规范（DOM 跨服同步染色定位）

当第三方业务流（无论是在云端、微信小程序还是远程网页）接收到上述 JSON 后，可以直接借助自带的 CDN 取到 HTML 以供极美展示。

为了保留 API 测试特有的“黄屏聚焦”，跨服前端需使用**跨模态寻址对齐策略**：

1. **Iframe 挂载**：直接将响应体中的 `html_view_url` 赋给原生 `<iframe src="...">`，利用浏览器内置引擎无损下载并渲染含数十万绝对定位元素的 `pdf2htmlEX` 页面。
2. **锚点提取**：取出 JSON 中的 `matched_block`，截取前后极度纯净的 20 个可见字符（去除空格、符号及特殊富文本标点），构成 `prefix` 和 `suffix`。
3. **脱水碰撞与涂色（TreeWalker Range）**：
    等 iframe `onload` 加载完毕，在前端执行一层探测脚本。将 HTML `<body>` 拍扁为纯文本流，通过 `indexOf(prefix)` 和 `indexOf(suffix, 起点)` 精确获得文本物理起点和终点！
    拉取 Web `Range` API，利用 `TreeWalker` 把框定范围内的所有远程物理节点强行包裹底色层（例如 `<span style="background-color:rgba(250,204,21,0.4)">`）。

通过这种 **CDN 派发文件 + 客户端动态脱水渲染** 的跨服架构：
*   **后端（Crawler）** 完全免去了跨网传输百兆 HTML 的巨大成本负担。
*   **大模型（LLM）** 仅接受极简纯净的 `.md` 文本或 URL，节省海量 Token 钱。
*   **人类用户** 依然能从远端获得拥有 100% 物理字体排版权威感的高定原装预览！
