# DPCrawler RAG Core Engine Deployment Guide

This package provides a standalone, zero-dependency deployment bundle for the DPCrawler RAG Engine.

## 1. Package Structure
After extracting the `.zip` file, you will find the following core components:
- `dpc_search_server` (`.exe` on Windows): The standalone backend executable.
- `dpc_search_server.html`: The HTML frontend interface for interactive testing.
- `README_DEPLOY.md`: This deployment guide.

*Note: Configuration files (e.g., `config.yaml`) and python environments are NOT required for deploying the core engine server.*

---

## 2. Standard Deployment (Portable Mode)

In Portable Mode, the server automatically maps its required resources based on the current working directory.

### Prerequisites
You must have a pre-crawled Data Directory (e.g., `output/`) generated by the crawler. 

### Deployment Steps
1. Create an empty directory on the target machine (e.g., `~/dpc_engine/`).
2. Move the extracted files (`dpc_search_server` and `dpc_search_server.html`) into this directory.
3. Copy your entire `output/` data directory into the same directory.

The directory structure must be aligned as follows:
```text
~/dpc_engine/
├── dpc_search_server      
├── dpc_search_server.html 
└── output/                
    └── example.com/
        ├── index.json
        └── html_views/
```

### Starting the Server
Run the executable directly from the terminal within the directory:
```bash
cd ~/dpc_engine/
./dpc_search_server -p 18088
```

---

## 3. Browser Usage

Once the server is running, you can access the frontend GUI via a web browser:

1. Open your browser and navigate to: `http://127.0.0.1:18088/search`
   *(If deploying on a remote server, replace `127.0.0.1` with your server's public IP address).*
2. **Site Name**: Click the dropdown to select a site name extracted from your dataset, or enter a domain identifier manually (e.g., `example.com`).
3. **Search Query**: Enter your keyword or sentence payload.
4. **Top K / Threshold**: Adjust the retrieving constraints.
5. Click **Search**. The results will appear below, providing direct previews and context highlighting for the mapped JSON data and HTML blocks.

---

## 4. Advanced: External Storage Mapping

If your `output/` directory is situated on a separate storage volume (e.g., `/mnt/nfs/big_data/output`), you do not need to move it into the binary directory.

You can run `./dpc_search_server` from any location. However, API requests querying the endpoint `POST /api/v1/search` must declare the overriding absolute physical path in the HTTP payload:

```json
{
    "site_name": "example.com",
    "query": "Machine Learning Guidelines",
    "top_k": 5,
    "output_dir": "/mnt/nfs/big_data/output"
}
```
